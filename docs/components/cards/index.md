## Формат карточки

Каждая карточка содержит:
- **Статус и владелец** компонента
- **Краткое описание** назначения
- **Основные интерфейсы** (API)
- **Ключевые метрики** производительности
- **Список зависимостей** (критичные и важные)
- **Основные возможности** компонента
- **Ссылка на детальную** документацию

## Использование

Карточки предназначены для:
- Быстрого понимания роли каждого компонента
- Изучения интерфейсов и зависимостей
- Определения метрик для мониторинга
- Навигации к детальной документации

## Ключевые принципы

- **Единообразие** - одинаковый формат для всех карточек
- **Краткость** - фокус на ключевой информации
- **Навигация** - ссылки на подробные документы
- **Актуальность** - отслеживание статуса и владельцев
